EntityAttributes: Code of Conduct and Jurisdiction
draft-macedir-entity-attribute-CoC-00.xml

Abstract

This document describes two SAML entity attributes that can be used by Entities in their SAML2 metadata. One can be used to indicate that an Entity is operating in compliance with a specific Code of Conduct; the second attribute states the legal Jurisdiction binding the Entity.

1.  Introduction and Motivation

This document describes a specific SAML "entity category attribute" [SAML2EntityAttribute] (Cantor, S., Ed., “SAML V2.0 Metadata Extension for Entity Attributes,” August 2009.) which indicates a claim that the entity thus labelled is committed to operating in compliance with a specific version of a Code of Conduct. It is expected that, over time, there may be multiple different Codes of Conduct; they may be developed by different communities, or may be newer versions of the developed by the same community. This specification is independent of whichever CoC is being referenced in a particular instance.

This document also describes a second SAML "entity category attribute" [SAML2EntityAttribute] (Cantor, S., Ed., “SAML V2.0 Metadata Extension for Entity Attributes,” August 2009.) whose value is an assertion of the country/legal jurisdiction which binds the Entity. This should indicate the legal framework under which the SP is operating, and under which disputes would be resolved if a contract does not exist or does not specify an alternative.

These SAML entity attribute claims MAY be used by a Service provider to convince an Identity Provider that the relying party is operating in compliance with the terms of the referenced Code of Conduct. Consequently an Identity Provider might consider the risk associated with releasing personal attribute values to that Service Provider to be acceptly low. This claim represents a statement, not a legal agreement.


2.  Entity Attribute of CoC Compliance


2.1.  Syntax

The value of the Name attribute on the enclosed SAML attribute is the identifier of the version of the Code of Conduct to which the Entity is committed. These values will typically be URIs. Typically, a Code of Conduct document will contain, within its text, the value to use as its "Name". There is no central Registry of these values.

The AttributeValue of the enclosed SAML attribute is a URL. Each individual Code of Conduct MAY specify what can be retrieved by resolving this url, and should also specify what steps, if any, an Identity Provider can take to process the retrieved document (eg to further validate compliance with the specified Code of Conduct).


2.2.  Semantics

The AttributeValue of the enclosed SAML attribute is a URL which resolves to a copy of the Code of Conduct identified by the Name attribute. The URL MUST resolve to the particular "version" of the Code of Conduct identified by the Name attribute; the text of the Code of Conduct MUST NOT be modified.


2.3.  Entity Example

<mdattr:EntityAttributes
          	xmlns:mdattr="urn:oasis:names:tc:SAML:metadata:attribute" >
        <Attribute xmlns="urn:oasis:names:tc:SAML:2.0:assertion"
        NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri"      Name="http://www.edugain.org/data-protection-code-of-conduct/sp-unilateral-01">
		 <AttributeValue>http://sp.example.org/my-signed-coc.png</AttributeValue>

        </Attribute>
        </mdattr:EntityAttributes>

3.  Entity Attribute of Jurisdiction


3.1.  Syntax

The Name of the enclosed SAML attribute is ''http://www.edugain.org/jurisdiction''.

The AttributeValue of the enclosed SAML attribute is an ISO 3166-1 alpha-2 code of the country, or, if that isn't sufficient to qualify the jurisdiction, an ISO 3166-2 code of the country and its subdivision (for instance, to indicate the state).


3.2.  Semantics

This EntityAttribute indicates that the country/jurisdiction that applies to the Entity if a contract does not exist or does not specify an alternative.


3.3.  Entity Support Example

<mdattr:EntityAttributes>
        <Attribute xmlns="urn:oasis:names:tc:SAML:2.0:assertion"
        NameFormat="urn:oasis:names:tc:SAML:2.0:attrname-format:uri"
        Name="http://www.edugain.org/jurisdiction">
            <AttributeValue>DE-BW</AttributeValue>
        </Attribute>
    </mdattr:EntityAttributes>

6.  Security Considerations

The presence of the Code of Conduct compliance entity attribute represents a series of claims (one for each attribute value) that the entity is committed to a Code of Conduct. The presence of the Jurisdiction entity attribute represents a claim that the entity is bound by the laws in a certain jurisdiction.

Before accepting and acting on such claims, any relying party needs to establish, at a level of assurance sufficient for the intended use, a chain of trust concluding that the claim is justified.

Some of the elements in such a chain of trust might include:

  • The integrity of the metadata delivered to the relying party, as for example assured by a digital signature.
  • If the entity category attribute is carried within a signed assertion, the assertion itself must be evaluated.
  • The procedures and policies of the immediate source of the metadata; in particular, any procedures the immediate source has with regard to aggregation of metadata from other sources.
  • The policies and procedures implemented by agents along the publication path from the original metadata registrar: this may be determined either by examination of the published procedures of each agent in turn, or may be simplified if the entity metadata includes publication path metadata as described in the [SAML2MetadataDRI] (La Joie, C., Ed., “SAML V2.0 Metadata Extensions for Registration and Publication Information Version 1.0,” April 2012.) extension.
  • The policies and procedures implemented by the original metadata registrar.
  • The definition of the Entity Attribute itself; in particular, any statements it makes about whether the value of the Entity Attribute may be self-asserted, or may only be asserted by particular authorities.
